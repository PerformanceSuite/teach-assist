## Implementation Details

### File Structure
```
data/
  helpContent.ts         # All articles and categories (16 articles, 6 categories)

components/
  HelpCenter/
    index.tsx            # Main Help Center component
    ContextualTip.tsx    # Contextual help tips (future)

stores/
  helpStore.ts           # Zustand store for help panel state

hooks/
  useGlobalShortcuts.ts  # Keyboard shortcuts including Cmd+/
```

### Search Functionality
- **Semantic Search:** Searches article titles, content, and tags
- **Live Filtering:** Results update as user types
- **Tag Matching:** Tags are searchable (e.g., "privacy", "grading")
- **Category Filtering:** Click category to see all articles in that category

### Keyboard Shortcuts
- **Cmd+/ (or Ctrl+/):** Toggle Help Center
- **Cmd+F:** Focus search field (when Help Center open)
- **Backspace:** Go back from article to list
- **Esc:** Close Help Center

### Article Viewing Tracking
- **Viewed Articles:** Tracked in localStorage via Zustand persist
- **"Viewed" Badge:** Shows on articles already read
- **Privacy:** View history never leaves the device

## Content Guidelines

### Article Writing Principles
1. **Teacher-First Language:** Write for teachers, not students or admins
2. **Practical Focus:** Every article should have actionable takeaways
3. **Example-Driven:** Use realistic teacher scenarios
4. **Honest About Limitations:** Be clear about what AI can/can't do
5. **Privacy-Conscious:** Always mention data protection considerations

### Tone
- **Professional but approachable**
- **Empowering, not patronizing**
- **Realistic about teacher workload**
- **Transparent about AI limitations**

### Structure
- **Clear headings** (use `**Heading**` format)
- **Short paragraphs** (3-4 lines max)
- **Bullet points** for lists
- **Examples** in real teaching contexts
- **Related articles** at the end
